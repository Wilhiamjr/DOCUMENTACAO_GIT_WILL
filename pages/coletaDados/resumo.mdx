# Armazenamento

As pastas nas quais todos os dados foram armazenados tinham uma estrutura aninhada conforme a seguir:

Dias (17 em total) -> Pessoas (5 por dia) -> Movimentos (18 por pessoa) -> Imagens (200 por movimento)  


Para facilitar o acesso e a transparência da estrutura as pastas foram nomeadas utilizando um sublinhado (underscore) seguido de uma notação numérica (*XX*), tal como:

<br/>
- [x] Cada pasta *Dia_XX* continha 5 pastas chamadas *Pessoa_XX*
- [x] Cada pasta *Pessoa_XX* continha 18 pastas chamadas *Movimento_XX*
- [x] Cada pasta *Movimento_XX* continha aproximadamente 200 imagens RGB e profundidade

A mesma estrutura de pastas foi sincronizada em tempo real com o armazenamento em nuvem (*pCloud*) que pôde ser rapidamente acessado e processado pela equipe de ciência de dados fora de Goiânia.
